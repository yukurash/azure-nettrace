赤信号

RF-04プライベートDNSゾーンが統合先VNetにリンクされていない
根拠
privatelink.database.windows.net は存在し、Aレコード contoso-sql → 10.20.2.4 も登録済み。だが contoso-vnet への仮想ネットワークリンクが 0 件。
影響
統合サブネットからは SQL の FQDN が公開IPに解決され、SQL は publicNetworkAccess=Disabled のため接続が確立できない（PEは承認済みだが迂回される）。
修正
az network private-dns link vnet create -g rg-contoso \
  -z privatelink.database.windows.net -n link-app \
  --virtual-network contoso-vnet --registration-enabled false
RF-06Key Vault が publicNetworkAccess=Disabled かつプライベートエンドポイント無し
根拠
contoso-kv は publicNetworkAccess=Disabled、networkAcls 未設定、PE 0 件。App Service の appSettings["KV_SECRET"] は @Microsoft.KeyVault 参照。
影響
公開経路もプライベート経路も断たれ、Key Vault 参照が解決されずアプリ設定値が空になる。※Bicep では Enabled 指定だが、テナントのポリシーが Disabled に上書きした実環境固有の欠陥。
修正
az network private-endpoint create -g rg-contoso -n pe-kv \
  --vnet-name contoso-vnet --subnet snet-pe \
  --private-connection-resource-id <kv-id> --group-id vault \
  --connection-name kvConn
# + privatelink.vaultcore.azure.net ゾーンを作成しVNetリンク
✅ 合格: RF-01（NSGは1433送信を拒否せず）· RF-07（PE承認済）· RF-08（SQL FW該当なし）· RF-11（vnetRouteAll=true）· RF-12（サブネット委任あり）
